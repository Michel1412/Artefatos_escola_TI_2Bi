Semana 3: 11/05

  • Elaboração do diagrama de classes de entidade, refinando os detalhes conforme necessário.

  • Elaboração do diagrama de entidade e relacionamento (DER), modelando as relações entre as entidades no banco de dados.

  • Organização dos projetos em três componentes principais: Front-end, Back-end e banco de dados.

  • Escolha das tecnologias para implementação:

    • Para o Front-end, “[utilizaremos React; especificamente, optaremos pelo Next.js para uma melhor experiência de desenvolvimento.]”

    • Para o Back-end, “[utilizaremos Node.js; optaremos pelo Nest.js para uma estrutura robusta e escalável.]”

    • Para o Banco de dados, “[escolheremos um modelo relacional devido à complexidade dos relacionamentos no sistema, garantindo melhor desempenho. Optaremos pelo PostgreSQL como tecnologia específica.]”

  • Sugestão: Criar repositórios no GitHub para o Front-end web, Front-end mobile e Back-end, facilitando a colaboração e versionamento do código.
