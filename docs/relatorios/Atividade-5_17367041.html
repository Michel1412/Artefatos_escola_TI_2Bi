Nome: Capturar Log de Erro

Atores:

  • Sistema Cliente (SDK)

  • Backend LogViewer

Descrição:
Recebe logs enviados pelo SDK e processa para e armazenamento.

Pré-condições:

  • SDK autenticado com token válido

  • Sistema do cliente cadastrado

Pós-condições:

  • Log armazenado no MongoDB

  • Log classificado e enriquecido com contexto

Fluxo Principal:

  1. SDK envia log via API segura

  2. Sistema valida token do SDK

  3. Sistema aplica mascaramento de dados sensíveis (RN-03)

  4. Sistema envia log para análise do LLM

  5. Sistema classifica o tipo de erro

  6. Sistema persiste o log no banco

  7. Sistema verifica necessidade de alerta

Fluxos Alternativos:
A1: Token inválido

  • Requisição é rejeitada

A2: Falha no LLM

  • Log é salvo sem análise

Regras de Negócio:

  • RN-03: Dados sensíveis devem ser mascarados

  • RN-02: Agrupar erros repetidos em 5 minutos
