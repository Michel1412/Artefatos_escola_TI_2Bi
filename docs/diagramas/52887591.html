Minha equipe de software : Classe de Caso de Uso -> Serviço de Tratamento do Log

Created by pedroalvaro193, last modified on mai. 25, 2026

TR-75 - Criar o diagrama de classe - 3 Concluído

Descrição: Classe responsável pelo processamento, enriquecimento e sanitização de logs brutos recebidos — seja via SDK instalada na aplicação cliente ou por varredura de diretório configurado. Atua como núcleo de transformação no Core Domain da arquitetura hexagonal, operando antes da persistência. Suas responsabilidades incluem: normalizar logs sem padrão definido (RF-07), aplicar o contexto empresarial do cliente para tipificação inteligente via LLM (RN-04), e — de forma obrigatória — validar e ofuscar credenciais ou senhas expostas nos logs antes de qualquer gravação no MongoDB (RN-03/RN-05). O resultado do tratamento é um documento estruturado com os campos obrigatórios: Timestamp, Tipo_do_Erro, Conteudo_do_Log e Contexto_Aplicado.
