Minha equipe de software : Diagrama de Sequência 3 -> Caso de uso - 3: Captura e Identificação de Logs Não Estruturados via SDK

title Fluxo de Captura e Processamento de Logs - LogViewer

actor "Sistema Cliente" as Client
participant "SDK LogViewer" as SDK
participant "Backend LogViewer" as Backend
database "MongoDB" as DB
participant "LLM/IA" as LLM
participant "Serviço de Notificação" as Notif

Client -> SDK: Gera Erro de Aplicação (Exception/Log)
activate SDK

note over SDK: Início do processamento local
SDK -> SDK: Capturar Erro Bruto Agnosticamente
SDK -> SDK: Mascarar Dados Sensíveis e Credenciais
SDK -> SDK: Anexar Token e Carimbo de Data/Hora

opt Se log > limite de tamanho
    SDK -> SDK: Gerenciar Logs Longos (Truncar/Chunking)
end

SDK -> Backend: Estabelecer Conexão Segura (Túnel Criptografado)
activate Backend

alt Se backend inalcançável (Falha de Conexão)
    Backend --x SDK: Timeout / Erro de Rede
    note over SDK: Ativação do «extend»\nGerenciar Falhas
    SDK -> SDK: Gerenciar Falhas de Conexão (Buffer Local)
else Conexão Bem Sucedida
    SDK -> Backend: Enviar Log Não Estruturado
    
    note over Backend: Validação e Persistência Inicial
    Backend -> Backend: Validar Token e Payload
    
    Backend -> DB: Persistir Log no Banco de Dados
    activate DB
    DB --> Backend: Confirmação de Persistência
    deactivate DB

    note over Backend: Processamento Avançado
    Backend -> Backend: Encaminhar para Processamento Avançado
    
    Backend -> LLM: Enviar payload para Análise
    activate LLM
    LLM --> Backend: Retornar Classificação e Contexto
    deactivate LLM
    
    Backend -> DB: Atualizar Log com insights da IA
    activate DB
    DB --> Backend: Confirmação de Atualização
    deactivate DB
    
    Backend -> Notif: Despachar Alertas Multicanal
    activate Notif
    Notif --> Backend: Confirmação de Envio
    deactivate Notif

    Backend --> SDK: Resposta de Sucesso (Ex: HTTP 202/200)
end

deactivate Backend
deactivate SDK
