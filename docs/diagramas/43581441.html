Caso de uso - 5: Gerenciar Assinatura e Processamento de Pagamento

Explicação Simplificada do Fluxo

Este diagrama ilustra a jornada de assinatura e pagamento dividida em três etapas lógicas:

  • 1. Consulta e Seleção de Planos: O Administrador acessa a tela de assinatura. O sistema primeiro verifica com o Clerk se ele realmente tem permissão. Em seguida, a API olha no Banco de Dados quantos sistemas aquele CNPJ possui para calcular e mostrar o preço correto do plano na tela.

  • 2. Processamento do Pagamento (Checkout): O usuário escolhe o plano e confirma a compra. O LogViewer envia essa cobrança para o Gateway de Pagamento (Mercado Pago, por exemplo). O Gateway tenta cobrar e devolve um status (pode aprovar na hora no cartão, ou ficar "Pendente" se for boleto/PIX). O sistema salva esse status e avisa o usuário.

  • 3. Confirmação Assíncrona (Webhook): Esta é uma ação que acontece "nos bastidores". Quando o dinheiro do cliente finalmente cai na conta (horas ou dias depois no caso de um boleto), o Gateway de Pagamento avisa o LogViewer automaticamente. A API verifica se a mensagem é segura e, por fim, muda o status da conta do cliente no banco de dados para "Ativo", liberando os acessos.
