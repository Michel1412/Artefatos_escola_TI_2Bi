Minha equipe de software : Classe de Caso de Uso ->API de Pagamento

Created by Vinicius Ferrarini on mai. 25, 2026

Descrição: Classe responsável por implementar a comunicação física (HTTP/REST) com o provedor externo de pagamentos (ex: Mercado Pago, Stripe, Abacate Pay). Na Arquitetura Hexagonal, ela atua como um Secondary Adapter (Adapter de Saída), implementando a interface (Port) definida pelo domínio. Sua responsabilidade primária é encapsular a complexidade técnica da integração (RF-28), gerenciando credenciais de API, timeouts, políticas de retry em caso de falha de rede e a transformação dos DTOs internos do LogViewer para o formato de JSON exigido pelo parceiro financeiro. Também possui a lógica criptográfica para validar se os webhooks recebidos são autênticos.
