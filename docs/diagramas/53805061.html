Minha equipe de software : Classe de Caso de Uso -> Serviço de Notificação

Created by pedroalvaro193, last modified on mai. 25, 2026

TR-75 - Criar o diagrama de classe - 3 Concluído

Descrição: Classe responsável por orquestrar e despachar alertas multicanais gerados a partir dos eventos de erro detectados pelo sistema LogViewer. Atua como Port de saída no domínio de notificações da arquitetura hexagonal, delegando o envio físico das mensagens para adapters específicos (e-mail, push mobile, WebSocket). Implementa obrigatoriamente a regra de fadiga de alertas (RN-02/RN-09): erros repetidos no intervalo de 5 minutos são agrupados em um único resumo, evitando spam. Notificações assíncronas são usadas para e-mail e push mobile (especialmente emergências), enquanto notificações síncronas via WebSocket alimentam os Toasts e Modais do Dashboard Web em tempo real (RN-08).
