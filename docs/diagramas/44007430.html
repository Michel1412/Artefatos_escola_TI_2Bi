title Receber e Aceitar Logs Agnósticos - LogViewer

actor "Sistema Cliente\n(SDK / Scanner)" as Client
participant "Backend LogViewer\n(Porta de Entrada / API)" as API
participant "Clerk\n(Gestor de Autenticação)" as Clerk
database "MongoDB\n(Adapter de Persistência)" as Mongo
queue "Message Broker\n(Fila de Processamento)" as MQ

Client->API: Enviar Log Bruto (Payload + Token)
activate API

alt Tamanho Payload > Limite
    note right of API: Gerenciar Excesso de Tamanho
    API-->Client: HTTP 413 Payload Too Large
else Tamanho dentro do limite
    
    note right of API: Validar Autenticidade do Token
    API->Clerk: Validar Token do Request
    activate Clerk
    Clerk-->API: Resultado da Validação
    deactivate Clerk

    alt Token Inválido
        note right of API: [Token Inválido]
        API-->Client: HTTP 401 Unauthorized
    else Token Válido
        
        note right of API: Identificar Sistema de Origem
        API->API: Extrair/Analisar Metadados da Origem
        
        alt Origem Não Localizada
            note right of API: Gerenciar Falha na Identificação
            API->API: Atribuir Origem Desconhecida / Falha
            API-->Client: HTTP 400 Bad Request
        else Origem Localizada
            
            note right of API: Mascarar Dados Sensíveis Básicos
            API->API: Aplicar máscaras (RN-05, RF-25)
            
            note right of API: Persistir Documento Bruto
            API->Mongo: Salvar RawLogEntry (Schema-less)
            activate Mongo
            Mongo-->API: Confirmação de Gravação
            deactivate Mongo
            
            note right of API: Enfileirar Log Bruto
            API->MQ: Publicar evento para Processamento Inteligente (IA)
            activate MQ
            MQ-->API: Confirmação de Enfileiramento
            deactivate MQ
            
            note right of API: Retornar HTTP Status 202
            API-->Client: HTTP 202 Accepted (Fire & Forget)
            
        end
    end
end

deactivate API
