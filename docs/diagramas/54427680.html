Minha equipe de software : Classe Principal -> UserToken

Created by Peng, last modified on mai. 25, 2026

Descrição

O UserToken representa o token de integração gerado por sistema cadastrado. Cada token é único por sistema (RN-02), com ciclo de vida controlado (geração, rotação e revogação). Ele é o elo entre o sistema do cliente e a SDK Java, garantindo isolamento de dados e rastreabilidade.

Atributos

Atributo

Tipo

Descrição

id

UUID

Identificador único do token

systemId

UUID

FK para o sistema ao qual pertence

cnpjOwnerId

String

CNPJ do dono (RN-01)

tokenHash

String

Hash do token (nunca armazenado em plain text)

tokenPrefix

String

Prefixo visível para identificação (ex: lgv_abc123)

status

TokenStatus

ACTIVE, REVOKED, EXPIRED

permissions

List<TokenPermission>

Escopos concedidos (ex: LOG_WRITE, SCAN)

createdAt

Instant

Data de criação

expiresAt

Instant

Data de expiração (nullable = sem expiração)

lastUsedAt

Instant

Último uso registrado

revokedAt

Instant

Data de revogação (nullable)

revokedBy

UUID

ID do usuário que revogou

Métodos

Método

Retorno

Descrição

generate(systemId, cnpj, permissions)

UserToken

Factory: cria e retorna token com hash gerado

rotate()

UserToken

Revoga o atual e gera novo token para o mesmo sistema

revoke(revokedByUserId)

void

Marca o token como REVOKED

isValid()

boolean

Verifica se status é ACTIVE e não expirado

hasPermission(permission)

boolean

Verifica se o token possui dado escopo

recordUsage()

void

Atualiza lastUsedAt com timestamp atual

toMaskedView()

TokenDTO

Retorna DTO com hash mascarado para exibição segura
