Minha equipe de software : Classe de Caso de Uso -> Serviço de Sugestão

Created by pedroalvaro193, last modified on mai. 25, 2026

TR-75 - Criar o diagrama de classe - 3 Concluído

Descrição: Classe responsável por mediar a interação entre o usuário e o LLM para geração de sugestões de solução vinculadas a erros persistidos. Atua como Port de saída para o adapter da Interface da LLM dentro da arquitetura hexagonal. Segue rigorosamente a RN-06 (Sugestão Passiva por Consentimento): o sistema nunca aplica correções automaticamente — toda sugestão gerada pela IA deve ser precedida de uma pergunta explícita de consentimento ao usuário ("Deseja receber uma possível solução para o erro X?"). Ao aceitar a sugestão, o status do erro é automaticamente alterado de Não Solucionado para Solucionado conforme a RN-07/RN-05. Também implementa a lógica de RF-19, identificando padrões recorrentes para sugestões proativas no Dashboard.
