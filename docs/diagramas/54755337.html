Minha equipe de software : Classe de Caso de Uso ->Processamento de Pagamento

Created by Vinicius Ferrarini, last modified on mai. 25, 2026

Descrição: Classe responsável por isolar e orquestrar a comunicação com gateways de pagamento externos (ex: Mercado Pago, Abacate Pay) dentro do Core Domain. Atua gerenciando o Outbound Port financeiro, garantindo o desacoplamento tecnológico exigido pela arquitetura hexagonal (RF-28). Gerencia tanto o fluxo síncrono (tentativa imediata de cobrança via cartão) quanto o fluxo assíncrono (conciliação de PIX/Boleto via Webhooks). Trabalha em conjunto com o Gerenciador de Assinatura, notificando-o para atualizar o status de acesso do cliente (RN-01) assim que um pagamento é confirmado, além de manter o histórico de transações (faturas) íntegro no banco de dados.
