Minha equipe de software : Classe: AuthenticationManager

Created by Michel Bocchi Junior on jun. 04, 2026

Descrição

Representa o componente central de gerenciamento de autenticação do sistema LogViewer.

É responsável por receber, validar, armazenar e distribuir tokens emitidos pelo Clerk, garantindo que os módulos internos utilizem informações autenticadas sem depender diretamente do provedor de identidade.

Centraliza a gestão de Access Tokens, Session Tokens, Claims JWT, Roles, Permissões e Metadados Customizados, funcionando como uma camada de abstração de segurança entre o Clerk, Backend API e demais componentes da aplicação.

Também é responsável por controlar renovação de sessões, validação de expiração e propagação de contexto autenticado para os serviços internos.

Na prática, o AuthenticationManager vira o guardião da sessão autenticada do LogViewer. Nenhum módulo precisaria conhecer JWT, Clerk, Claims ou Metadata diretamente; todos consumiriam apenas o contexto autenticado fornecido por ele. Isso normalmente é o desenho utilizado em plataformas SaaS multi-tenant com RBAC, exatamente o cenário que seu LogViewer está seguindo.
