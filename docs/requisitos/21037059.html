TR-27 - Criação de caso de uso - 10: Configurar e Gerenciar Contexto Concluído

Itens associados:

Requisitos Funcionais:

  • RF-08: O Usuário deve configurar um Contexto detalhando a área de atuação de sua empresa.

  • RF-09: O Sistema aplicará o contexto configurado no momento da varredura.

  • RF-11: O Usuário poderá alterar o contexto configurado a qualquer momento.

Regras de Negócio:

  • RN-04: O contexto deve ser aplicado na análise dos logs

  • RN-01: Contexto está vinculado ao CNPJ

Nome: Configurar e Gerenciar Contexto

  • Atores:

    • Usuário.

    • Serviço de LLM (IA).

Descrição:

  • Permite ao usuário definir, atualizar e redefinir o contexto da empresa, que será utilizado na análise inteligente dos logs.

Pré-condições:

  • Usuário Autenticado.

  • Conta Ativa.

Pós-condições:

  • Contexto salvo e disponível para uso da análise de Logs.

Fluxo Principal:

  • Usuário acessa a área de configurações

  • Sistema exibe formulário de contexto

  • Usuário informa:

    • Ramo de atuação.

    • Stack tecnológica.

    • Nível de criticidade.

  • Usuário salva configuração.

  • Sistema persiste o contexto.

  • Sistema associa o contexto aos sistemas do cliente.

Fluxo Alternativo:

  • A1: Atualização de contexto existente

    • Usuário edita informações.

    • Sistema sobrescreve o contexto anterior.

  • A2: Alteração de contexto impactando logs futuros

    • Sistema aplica novo contexto apenas para novos logs.

Regras de Negócio:

  • RN-04: O contexto deve ser aplicado na análise dos logs.

  • RN-01: Contexto está vinculado ao CNPJ.

Requisitos:

  • RF-08.

  • RF-11.

  • RF-09.

Observações:

  • Alterações de contexto não reprocessam logs antigos automaticamente.

  • Pode existir feature futura de reprocessamento.
