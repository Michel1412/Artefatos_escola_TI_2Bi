Minha equipe de software : Equipe 1

Created by João Pedro Souza Peixoto Saraiva, last modified by Vinicius Ferrarini on mar. 12, 2026

  • RF - 15: O Sistema deve gerir o Usuário

  • RF - 16: O Sistema deve gerencias tokens para a integração SDK.

  • RF - 17: O Sistema deve ter um Painel com Dashboard.

  • RF - 18: O Sistema deve conter um gerenciamento de assinatura do usuário

  • RF - 19: O Sistema vai sugerir soluções para problemas recorrentes

  • RF - 20: O Sistema deve conter uma página de relatório para os multi-sistemas

  • RF - 21: O Sistema deve ter uma barra de pesquisa dos erros encontrado já armazenado

  • RF - 22: O Sistema tem que ter um contador de logs repetidos

  • RF - 23: O Sistema tem que ter a opção Status por erro Não solucionado, Solucionado

  • RF - 24: O Sistema tem que ter a opção de exportar para PDF e Excel

  • RF - 25: O Sistema tem que validar se tem senhas vazadas nos logs

  • RF - 26: O Sistema tem que existir uma maneira de confirmação de leitura

  • RF - 27: O Sistema deve poder arquivar erros

  • RF - 28: O Sistema deve ter uma integração com uma API de pagamento (ex: Mercado Pago, abacate pay)

  • RF - 29: O Sistema deve uma forma de catalogar os erros(marcar pendente)

  • RN - 01: Todo usuário é um CNPJ, também deve conter uma lista de sistema, deve ser único , deve aumentar o preço a cada adição de sistema, alterar os dados. Os tokens da SDK deve ser único por sistema.(RF - 15, RF-16, )

  • RN - 02: O plano de pagamento do sistema pode ser fixo ou negociavel, integrado com plataforma de pagamento(RF-18 - 28)

  • RN - 03: O dashboard do cliente apresentar dados processados pelo sistema, também deve apresentar sugestões para os problemas encontrados recorrentes, tem que ter um filtro para multi sistemas, deve ser capaz de exportar para PDF e Excel(17 - 19 - 20 - 22 -24)

  • RN - 04: A barra de pesquisa deve ser capaz de encontrar todos os erros registrados.

  • RN - 05: O Sistema tem que ter a opção Status por como Não solucionado, Solucionado,a o solicitar uma solução para LLM o status deve ser alterado para solucionado. Os marcadores serão de escolha do usuário com base nas opções existentes, Pendente, Arquivado, Ignorar etc e também pode ser marcadores personalizados(como marcadores do Email) (29 - 23 - 26 - 27)

  • RN - 06: O sistema deve validar credencias vazadas nos logs.
