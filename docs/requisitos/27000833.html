Minha equipe de software : Caso de uso - 21: Visualizar Dashboard Interativo

Created by Peng, last modified on abr. 02, 2026

Nome: Visualizar Dashboard Interativo

Atores:

Usuário (CNPJ autenticado)
Backend LogViewer

Descrição:
Permite ao usuário visualizar um painel interativo com dados consolidados de erros, métricas, tendências, alertas em tempo real e sugestões inteligentes baseadas nos logs capturados.

Pré-condições:

Usuário autenticado (RF-01)
Usuário possui pelo menos um sistema cadastrado (RN-01)
Logs já processados e armazenados no MongoDB (RN-04)

Pós-condições:

Dashboard renderizado com dados atualizados
Eventos de erro exibidos com filtros aplicados
Alertas em tempo real apresentados via WebSocket

Fluxo Principal:

Usuário acessa o painel Dashboard via aplicação Web ou Mobile

Sistema valida autenticação e permissões do usuário

Sistema consulta os sistemas vinculados ao CNPJ

Sistema aplica filtro padrão (multi-sistemas ou sistema específico) (RN-10)

Sistema busca dados consolidados no MongoDB:

  • Quantidade de erros

  • Tipos de erro

  • Frequência (logs repetidos) (RF-22)

  • Status (solucionado/não solucionado) (RF-23)

Sistema processa agregações para métricas e gráficos

Sistema identifica padrões e problemas recorrentes (RF-19)

Sistema exibe sugestões inteligentes baseadas em recorrência

Sistema inicia conexão WebSocket para eventos em tempo real (RN-08)

Sistema renderiza dashboard com:

  • Gráficos (tempo, tipo de erro, criticidade)

  • Lista de erros recentes

  • Indicadores de criticidade

  • Filtros dinâmicos

Sistema permite interação do usuário:

  • Aplicar filtros (RF-14)

  • Pesquisar logs (RF-21)

  • Navegar para detalhes do erro

  • Solicitar solução via LLM (RN-06)

Fluxos Alternativos:

A1: Usuário sem sistemas cadastrados

Sistema exibe estado vazio com orientação para cadastro de sistema

A2: Falha na consulta ao banco

Sistema exibe mensagem de erro
Sistema permite tentativa de recarregamento

A3: Falha na conexão WebSocket

Dashboard carrega sem dados em tempo real
Sistema tenta reconexão automática

A4: Alto volume de dados

Sistema aplica paginação e lazy loading
Sistema prioriza dados mais recentes

Regras de Negócio:

RN-01: Usuário pode visualizar múltiplos sistemas no mesmo dashboard

RN-04: Todos os logs exibidos devem estar classificados e com contexto aplicado

RN-08: Dashboard deve receber atualizações em tempo real via WebSocket

RN-09: Logs repetidos devem ser agrupados para evitar poluição visual

RN-10: Dashboard deve permitir visualização por sistema ou multi-sistemas

RN-06: Sistema apenas sugere soluções, nunca executa automaticamente
