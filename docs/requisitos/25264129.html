Minha equipe de software : Caso de uso - 9: Interagir com Agente IA para Solução

Created by Peng on mar. 30, 2026

Nome: Interagir com Agente IA para Solução

Atores:

  • Usuário (Gestor/Desenvolvedor)

  • Backend LogViewer (Orquestrador)

  • Agente LLM (Interface de IA)

Descrição: O sistema disponibiliza um link ou ambiente de chat onde o Usuário pode interagir com um Agente IA que já possui o contexto completo do erro (Stack, Ramo, Criticidade e Conteúdo do Log) para obter diagnósticos e sugestões de correção.

Pré-condições:

  • Log de erro já processado e persistido no MongoDB.

  • Contexto da empresa previamente configurado (RN-04).

  • Usuário autenticado e com permissão de visualização do log.

Pós-condições:

  • Interface de chat com a IA aberta com o histórico do erro carregado.

  • Status do erro alterado para "Solucionado" caso o usuário aceite a proposta (RN-07).

Fluxo Principal:

  1. O Usuário visualiza um erro específico no Histórico ou via Alerta.

  2. O sistema apresenta a sugestão passiva: "Deseja receber uma possível solução para o erro X?" (RN-06).

  3. O Usuário clica no botão/link para interagir com o Agente.

  4. O Backend gera um "Payload de Contexto" contendo: Stack tecnológica, Ramo de atuação do cliente, Conteúdo do Log e o Timestamp (RN-04).

  5. O sistema redireciona o usuário para o ambiente do Agente, injetando o contexto automaticamente via API.

  6. O Usuário interage com o Agente para tirar dúvidas ou solicitar código de correção.

  7. O Usuário confirma que a solução é satisfatória.

  8. O sistema altera o status do erro de "Não Solucionado" para "Solucionado" automaticamente (RN-07).

Fluxos Alternativos:

  • A1: Usuário ignora a sugestão

    1. O sistema mantém o link disponível, mas não abre a interface da IA. O status permanece "Não Solucionado".

  • A2: Falha na recuperação do contexto

    1. O sistema informa que o contexto original foi perdido ou alterado e abre o Agente apenas com os dados brutos do log.

Regras de Negócio Aplicadas:

  • RN-06: O sistema não corrige o erro sozinho; ele depende do consentimento do usuário para sugerir a solução.

  • RN-07: A aceitação da solução via interface do Agente implica na mudança automática de status do log no banco de dados.
