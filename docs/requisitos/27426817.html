Minha equipe de software : Caso de uso - 22: Integração e Visualização de Logs via SDK Java e Dashboard React

Created by Peng on abr. 02, 2026

Nome: Integração e Visualização de Logs via SDK Java e Dashboard React

Atores:

  • Sistema Cliente (SDK Java)

  • Backend LogViewer

  • Usuário (via Dashboard Web React)

Descrição:

Permite que aplicações cliente enviem logs através de uma SDK Java padronizada, enquanto o usuário visualiza, em tempo real, esses logs processados em um dashboard Web desenvolvido em React (ou React Native Web), garantindo consistência de interface e performance.

Pré-condições:

  • SDK Java instalada e configurada no sistema cliente

  • Token de integração válido e vinculado a um sistema (RN-02)

  • Usuário autenticado no dashboard Web

  • Sistema cadastrado com contexto definido

Pós-condições:

  • Logs capturados via SDK Java são exibidos no Dashboard React

  • Dados apresentados de forma padronizada e performática

  • Interface consistente entre Web e Mobile (quando aplicável)

Fluxo Principal:

  1. Aplicação cliente inicializa a SDK Java com o token do sistema

  2. SDK captura logs de erro automaticamente ou via interceptadores

  3. SDK envia logs para o Backend via API segura (HTTPS + criptografia)

  4. Backend valida o token e identifica o sistema

  5. Backend processa e persiste o log no MongoDB

  6. Backend publica evento via WebSocket

  7. Dashboard React recebe evento em tempo real

  8. Interface atualiza dinamicamente exibindo o novo log

  9. Usuário visualiza o erro no dashboard com filtros e contexto

Fluxos Alternativos:

A1: SDK não inicializada corretamente

  • Logs não são capturados

  • Sistema cliente registra erro localmente

A2: Incompatibilidade de versão da SDK

  • Backend rejeita payload

  • SDK deve ser atualizada

A3: Falha na conexão WebSocket

  • Dashboard realiza fallback para polling

Regras de Negócio Aplicadas:

  • RNF-02: SDK deve ser inicialmente desenvolvida em Java, com arquitetura extensível para outras linguagens (ex: Node, Python)

  • RNF-10: Frontend Web deve utilizar React (ou React Native Web) garantindo padronização e reaproveitamento entre Web e Mobile
