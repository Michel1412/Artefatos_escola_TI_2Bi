Requisitos Não Funcionais

  • RN - 01: A conexão do SDK com o Core deve ser criptografada.

  • RN - 02: A primeira versão do SDK deve ser em JAVA

  • RN - 03: O projeto deve estimar outras versão de SDKs

  • RN - 04: Debater sobre utilizar um Sistema externo de gestão de usuários

  • RN - 05: Debater sobre o tipo de Banco de dados

  • RN - 06: Escolher qual banco de dados atende melhor às necessidades

  • RN - 08: Escolher a arquitetura do Sistema.

  • RN - 09: O Sistema deve salvar logs temporariamente

  • RN - 10: Disponibilizar um Webhook para os SDKs

  • RN - 11: O Sistema deve ser capaz de varrer qualquer arquivo que armazenam historicos de Logs.

  • RN - 12: Utilizar padrões de projeto para melhorar a qualidade e eficiência

  • RN - 13: O aplicativo e o sitema web devem utilizar dados otimizados, visando reduzir o PLT (Page Load Time)

Regras de negócio Parte 2

  • RB - O nosso sistema deve notificar situações emergenciais através do aplicativo mobile

  • RB - O aplicativo mobile deve conter um dashboard simplificado

  • RB - O aplicativo mobile deve conter um chat para interagir com o LLM

  • RB - A criptografia do SDK deve ser por túnel de criptografia

  • RB - Disponibilizar SDKs em diversas linguagens, inicialmente por Java

  • RB - O sistema utilizará de um método de gestão de usuários com a plataforma Clerk, visando maior segurabilidade, facilidade no desenvolvimento e escalabilidade

  • RB - O sistema utilizará o MongoDB como gerenciador de banco de dados, visando a maleabilidade dos dados

  • RB - O sistema utilizará a arquitetura de sistema hexagonal

  • RB - O sistema terá um plano com disponibilidade de salvamento de logs de forma temporária durante 7 dias

  • RB - O sistema terá uma API voltada em desempenho e velocidade para o não travamento do SDK, utilizando do webhook

  • RB - O sistema poderá receber multiplos formatos de arquivos de logs

  • RB - O desenvolvimento será orientado a padrões de projeto visando uma implementação mais rápida e eficiente

  • RB - O Sistema Web deve ser programado em React Native, visando facilitar o desenvolvimento pela equipe

  • RB - O aplicativo mobile deve ser programado em React Native, visando facilitar o desenvolvimento pela equipe

  • RB- Os projetos Mobile e Web serão divididos para melhor gestão da equipe

  • RB - Os usuários que tiverem autenticação de dois fatores precisarão ter o aplicativo mobile instalado
