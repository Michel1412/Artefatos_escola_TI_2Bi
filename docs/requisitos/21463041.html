Minha equipe de software : Caso de uso - 1: Notificar Erros Encontrados (RF-04).

Created by João Pedro Souza Peixoto Saraiva, last modified on abr. 16, 2026

Itens associados: Requisitos Funcionais:

  • RF-04: O Sistema deve notificar os erros encontrados em múltiplos canais.

  • RF-05: O Sistema deve alertar possíveis situações críticas descobertas pelo SDK.

Nome: Notificar Erros Encontrados Atores:

  • Sistema (Worker de Processamento de Logs).

  • Provedores de Email (ex: SendGrid).

  • Provedores de Push Notification (ex: Firebase/FCM).

  • Servidor de WebSockets.

Descrição: Gerencia o roteamento multicanal da notificação de erros e alerta sobre possíveis situações críticas descobertas pelo SDK. É acionado imediatamente após um log de erro ou alerta crítico ser capturado, contextualizado pela IA, ofuscado (prevenção de dados sensíveis) e persistido no banco de dados.

Pré-condições:

  • O log de erro ou alerta do SDK deve estar persistido no banco de dados (MongoDB) contendo Timestamp, Tipo_do_Erro, Conteudo_do_Log e Contexto_Aplicado.

Pós-condições:

  • Notificações e alertas críticos enviados aos usuários (Dashboard, Push, E-mail) ou agrupados para envio posterior em caso de detecção de spam/fadiga.

Fluxo Principal:

  1. Sistema recebe a entidade do Erro ou alerta crítico recém-persistido vindo do SDK.

  2. Sistema despacha um evento síncrono via WebSocket contendo o payload do erro para exibição de Toasts/Modais no Dashboard Web em tempo real, destacando se for uma situação crítica.

  3. Sistema despacha um evento assíncrono de Push Notification para os dispositivos móveis atrelados aos usuários do sistema.

  4. Sistema despacha um evento assíncrono de E-mail, detalhando a criticidade do erro ou da situação descoberta pelo SDK e sugerindo o acesso ao Agente IA.

Fluxo Alternativo: A1: Prevenção de Fadiga de Alertas

  1. Sistema silencia os canais assíncronos (aborta o envio imediato de E-mail e Push Notification para evitar spam).

  2. Sistema adiciona o erro a uma fila de "Resumo de Agrupamento".

  3. Sistema atualiza o Dashboard Web via WebSocket silenciosamente (apenas incrementando o badge de contagem na UI, sem disparar um novo Toast).
