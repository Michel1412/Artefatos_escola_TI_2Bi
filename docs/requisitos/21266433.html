Caso de Uso: Autenticar Usuário

Nome: Autenticar Usuário Atores:

  • Usuário (Gestor/Desenvolvedor do cliente)

  • Sistema de Autenticação (Clerk)

  • Aplicativo Mobile LogViewer (Aprovador 2FA)

  • Backend LogViewer

Descrição: Realiza a identificação e validação do usuário no sistema, garantindo acesso seguro ao dashboard web ou mobile, aplicando o segundo fator de autenticação quando configurado.

Pré-condições:

  • Usuário previamente cadastrado no sistema (Vinculado a um CNPJ - RN-01).

  • Conexão estável com a internet.

Pós-condições:

  • Token de sessão JWT (JSON Web Token) gerado e armazenado com segurança.

  • Acesso liberado às funcionalidades baseadas no plano de assinatura.

Fluxo Principal:

  1. O Usuário insere suas credenciais (E-mail/Senha ou Social Login) no Portal Web ou App Mobile.

  2. O sistema valida as credenciais via Clerk Adapter.

  3. O sistema verifica se o perfil do usuário exige MFA/2FA (RN-03).

  4. Se 2FA Ativo: O sistema envia uma notificação push para o dispositivo mobile vinculado.

  5. O Usuário aprova o acesso no Aplicativo Mobile (usando FaceID/TouchID - RF-34).

  6. O sistema confirma a validade da assinatura do CNPJ (RN-01/RF-18).

  7. O sistema libera o acesso ao Dashboard com o contexto do cliente.

Fluxos Alternativos:

  • A1: Credenciais Inválidas

    1. O sistema exibe mensagem de erro genérica ("E-mail ou senha incorretos").

    2. O acesso é bloqueado.

  • A2: 2FA Ativo mas App Mobile não instalado

    1. O sistema bloqueia o login e instrui o usuário a instalar o app para validar o acesso (Conforme RN-03).

  • A3: Assinatura Inativa/Inadimplente

    1. O sistema autentica o usuário, mas redireciona para a página de gestão de assinatura (RF-18).

Regras de Negócio Aplicadas:

  • RN-01: O login deve identificar o CNPJ vinculado para carregar os sistemas corretos.

  • RN-03: Exigência estrita de aprovação via App Mobile se o 2FA estiver habilitado no perfil.
